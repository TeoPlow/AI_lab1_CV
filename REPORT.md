# Отчет по Лабораторной работе 1 (CV)

## 1. Постановка задачи
Провести исследование модели обнаружения объектов для практической задачи контроля безопасности на производстве.

## 2. Начальные условия
- Датасет: Hard Hat Workers, версия v8 (raw_HelmetClassOnly)
- Класс: `helmet`
- Формат данных: YOLO

## 3. Выбор метрик
- mAP50-95
- mAP50
- Precision
- Recall

Обоснование:
- mAP50-95 как главная интегральная метрика;
- mAP50 для удобного сравнения запусков;
- Precision/Recall для баланса ложных срабатываний и пропусков.

## 4. Эксперименты

Результаты запусков:

| Модель | Epochs | imgsz | mAP50-95 | mAP50 | Precision | Recall |
|---|---:|---:|---:|---:|---:|---:|
| yolo11n | 30 | 640 | 0.6942 | 0.9820 | 0.9585 | 0.9405 |
| yolo11s | - | - | - | - | - | - |

Фактический запуск:
- train run dir: runs/train/helmet_detector_gpu_final
- best weights: runs/train/helmet_detector_gpu_final/weights/best.pt
- test metrics file: runs/train/helmet_detector_gpu_final/metrics_summary.json
- длительность обучения: 30 эпох за 0.462 часа (около 27.7 минут)

## 5. Примеры предсказаний
Примеры для вставки в отчет:
- запуск инференса с лучшими весами: `python src/lab1_yolo11.py predict --weights runs/train/helmet_detector_gpu_final/weights/best.pt --source dataset/test/images --device 0`
- изображения после инференса: папка `runs/predict/helmet_detector_gpu_final`

Комментарий по качеству детекции:
- модель уверенно детектирует каски на большинстве кадров;
- основная часть ошибок связана с мелкими/частично перекрытыми объектами и сложными ракурсами;
- качество достаточно высокое для базового мониторинга в задаче на тройку.

## 6. Вывод
По результатам эксперимента `yolo11n` на датасете Hard Hat Workers (v8, класс `helmet`) достигнуты:
- mAP50 = 0.9820;
- mAP50-95 = 0.6942;
- Precision = 0.9585;
- Recall = 0.9405.

Выводы:
- модель `yolo11n` обеспечивает хороший баланс скорости и качества для базового сценария обнаружения;
- наиболее вероятные ошибки возникают на маленьких и частично закрытых объектах;
- для улучшения качества можно добавить эксперимент с `yolo11s`, увеличить `imgsz`, а также использовать дополнительные аугментации.

Итог по объему работ:
- требования варианта на тройку выполнены: выбраны и обоснованы начальные условия, реализовано обучение/оценка/инференс на Ultralytics YOLOv11, получены и зафиксированы итоговые метрики.
